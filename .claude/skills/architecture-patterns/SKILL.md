# Architecture & Design Skills

## Architecture Patterns
- Layered architecture, hexagonal architecture, clean architecture
- Microservices vs monolith decision framework
- Event-driven architecture, CQRS, Event Sourcing
- Domain-Driven Design (DDD) tactical patterns
- Capacity planning and scaling patterns
- CAP theorem tradeoff analysis
- Caching strategies (CDN, Redis, application cache)

## Architecture Decision Records (ADR)
- Document architectural decisions with context and rationale
- Template: Title, Status, Context, Decision, Consequences
- Track rejected alternatives with reasoning
- Keep ADRs in version control alongside the codebase

## Improve Codebase Architecture (Matt Pocock)
- Find deepening opportunities in the codebase
- Identify shallow modules that need restructuring
- Create architectural improvement plans with visual diagrams
- Balance refactoring cost against future maintenance benefits
